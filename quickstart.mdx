## Prerequisites

Before you begin, ensure you have the following installed:

- **Node.js 20+** - [Download](https://nodejs.org/)
- **Docker Desktop** (recommended) - [Download](https://www.docker.com/products/docker-desktop/)
- **Git** - [Download](https://git-scm.com/)

## Installation Steps

<Steps>
  <Step title="Clone the Repository">
    Clone the SMyEG repository to your local machine:
    
    ```bash
    git clone https://github.com/drosales10/SMyEG.git
    cd SMyEG
    ```
  </Step>
  
  <Step title="Install Dependencies">
    Install the required npm packages:
    
    <CodeGroup>
    ```bash npm
    npm install
    ```
    
    ```bash yarn
    yarn install
    ```
    
    ```bash pnpm
    pnpm install
    ```
    </CodeGroup>
  </Step>
  
  <Step title="Start Database Services">
    Launch PostgreSQL and Redis using Docker Compose:
    
    ```bash
    docker compose up -d
    ```
    
    This will start:
    - PostgreSQL 15 on port `5432`
    - Redis 7 on port `6379`
  </Step>
  
  <Step title="Configure Environment">
    Create your environment file from the template:
    
    ```bash
    cp .env.example .env
    ```
    
    The default `.env` should work out of the box with the Docker setup. Key variables:
    
    ```bash .env
    DATABASE_URL="postgresql://postgres:postgres@localhost:5432/modular_app"
    NEXTAUTH_SECRET="your-secret-key-here"
    NEXTAUTH_URL="http://localhost:3000"
    ```
    
    <Warning>
    Change `NEXTAUTH_SECRET` to a secure random string in production.
    </Warning>
  </Step>
  
  <Step title="Initialize the Database">
    Generate Prisma client and apply migrations:
    
    ```bash
    npm run db:generate
    npm run db:migrate
    ```
    
    Load seed data (including default admin user):
    
    ```bash
    npm run db:seed
    ```
  </Step>
  
  <Step title="Start the Development Server">
    Launch the Next.js application:
    
    ```bash
    npm run dev
    ```
    
    The application will be available at [http://localhost:3000](http://localhost:3000)
  </Step>
  
  <Step title="Login with Default Credentials">
    Navigate to [http://localhost:3000](http://localhost:3000) and login with:
    
    <Card title="Default Admin Account" icon="user-shield">
      **Email:** `admin@example.com`  
      **Password:** `Admin1234`
    </Card>
    
    <Warning>
    Change the default admin password immediately after first login!
    </Warning>
  </Step>
</Steps>

## Verify Installation

Once logged in, you should see the SMyEG dashboard with:

- User management interface
- Organization selector (if you're part of multiple organizations)
- Forest patrimony management (if the module is enabled)
- System configuration options

## Optional: Start Background Workers

SMyEG includes background workers for geospatial processing and asset measurements. To start them:

<Tabs>
  <Tab title="Geo Worker">
    Process shapefile imports and geometry calculations:
    
    ```bash
    npm run worker:geo
    ```
    
    For a single run (useful for testing):
    
    ```bash
    npm run worker:geo:once
    ```
  </Tab>
  
  <Tab title="Asset Worker">
    Process biological asset measurements and calculations:
    
    ```bash
    npm run worker:assets
    ```
    
    For a single run:
    
    ```bash
    npm run worker:assets:once
    ```
  </Tab>
</Tabs>

<Info>
Workers are optional for basic functionality but required for shapefile imports and measurement calculations.
</Info>

## Common Issues

<AccordionGroup>
  <Accordion title="Database connection failed">
    Ensure Docker containers are running:
    
    ```bash
    docker compose ps
    ```
    
    If containers are not running, start them:
    
    ```bash
    docker compose up -d
    ```
  </Accordion>
  
  <Accordion title="Port 3000 already in use">
    Change the port in your `package.json`:
    
    ```json package.json
    {
      "scripts": {
        "dev": "next dev -p 3001"
      }
    }
    ```
  </Accordion>
  
  <Accordion title="Prisma migration errors">
    Reset the database and reapply migrations:
    
    ```bash
    npm run db:push
    npm run db:seed
    ```
    
    <Warning>
    This will delete all existing data!
    </Warning>
  </Accordion>
</AccordionGroup>
