## Overview

SMyEG includes comprehensive forest-specific configuration types defined in the Prisma schema. These enumerations standardize forest patrimony hierarchies, biological assets, and measurement protocols.

## Forest Patrimony Hierarchy

### Level 2 Types (PatrimonyLevel2Type)

Top-level forest patrimony units representing properties or reserves:

| Type | Description |
|------|-------------|
| `FINCA` | Farm or estate |
| `PREDIO` | Land parcel or property |
| `HATO` | Ranch or cattle farm |
| `FUNDO` | Rural property or estate |
| `HACIENDA` | Large estate or plantation |
| `ABRAE` | Protected natural area (Venezuela) |

```prisma
enum PatrimonyLevel2Type {
  FINCA
  PREDIO
  HATO
  FUNDO
  HACIENDA
  ABRAE
}
```

### Level 3 Types (PatrimonyLevel3Type)

Intermediate subdivisions within Level 2 units:

| Type | Description |
|------|-------------|
| `COMPARTIMIENTO` | Compartment |
| `BLOCK` | Block division |
| `SECCION` | Section |
| `LOTE` | Lot or batch |
| `ZONA` | Zone |
| `BLOQUE` | Block (alternative) |
| `ZONIFICACION` | Zoning area |

```prisma
enum PatrimonyLevel3Type {
  COMPARTIMIENTO
  BLOCK
  SECCION
  LOTE

```

## Plot and Geometry Configuration

### Plot Shape Types

Standard shapes for Level 5 sampling plots:

| Shape | Description | Use Case |
|-------|-------------|----------|
| `RECTANGULAR` | Rectangular plot | Standard inventory plots |
| `CUADRADA` | Square plot | Fixed-area sampling |
| `CIRCULAR` | Circular plot | Variable-radius plots |
| `HEXAGONAL` | Hexagonal plot | Systematic sampling grids |

```prisma
enum PlotShapeType {
  RECTANGULAR
  CUADRADA
  CIRCULAR
  HEXAGONAL
}
```

### Dimensions Storage

Level 5 units store 4 dimensions based on shape:

- **Rectangular:** `dimension1M` (length), `dimension2M` (width)
- **Circular:** `dimension1M` (radius)
- **Square:** `dimension1M` (side length)
- **Hexagonal:** `dimension1M` (side length)

### IMA Classification

Mean Annual Increment (IMA) quality classes:

| Class | Description | Productivity |
|-------|-------------|-------------|
| `I` | Class I | Highest productivity |
| `II` | Class II | High productivity |
| `III` | Class III | Medium productivity |
| `IV` | Class IV | Low productivity |
| `V` | Class V | Lowest productivity |

```prisma
enum ImaClassification {
  I
  II
  III
  IV
  V
}
```

<Info>
  IMA classes are typically defined with range thresholds (e.g., Class I: >30 m³/ha/year) stored in the `ImaClass` catalog table.
</Info>

## Usage Examples

### Creating a Level 2 Unit

```typescript
const level2 = await prisma.forestPatrimonyLevel2.create({
  data: {
    organizationId: 'org-uuid',
    code: 'FINCA-001',
    name: 'Finca El Bosque',
    type: 'FINCA',
    legalStatus: 'ADQUISICION',
    totalAreaHa: 1500.50,
    centroidLatitude: 10.5,
    centroidLongitude: -67.5,
  },
});
```

### Creating a Measurement Campaign

```typescript
const campaign = await prisma.assetMeasurementCampaign.create({
  data: {
    organizationId: 'org-uuid',
    level4Id: 'level4-uuid',
    domain: 'BIOLOGICO',
    protocolType: 'BIO_SOBREVIVENCIA',
    campaignCode: 'SURV-2026-03',
    campaignName: 'Survival Monitoring March 2026',
    campaignDate: new Date('2026-03-15'),
    status: 'DRAFT',
  },
});
```
