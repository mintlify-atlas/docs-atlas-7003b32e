# Introduction to SMyEG

SMyEG is a comprehensive full-stack modular application platform designed for building enterprise-grade systems with authentication, authorization, user management, and extensive configuration capabilities.

## What is SMyEG?

SMyEG is a production-ready application framework that combines modern web technologies to deliver a scalable, secure, and maintainable platform for building business applications. Built with Next.js, React, and PostgreSQL, it provides a solid foundation for developing complex enterprise solutions.

## Key Capabilities

### Authentication & Authorization
- NextAuth v5 integration with credential-based authentication
- Multi-organization support with tenant isolation
- Role-based access control (RBAC) with granular permissions
- Support for multiple authentication providers (Local, Google, GitHub, Microsoft)

### User Management
- Complete user lifecycle management (active, inactive, pending verification, locked)
- Flexible role assignment with system and custom roles
- User profile management and verification workflows
- Organization-scoped user administration

### Modular Architecture
SMyEG includes the following pre-built modules:

- **Authentication & Authorization** - Secure login and access control
- **User Management** - Complete user administration
- **Dashboard & Metrics** - Analytics and reporting
- **System Configuration** - Global and organization-specific settings
- **Audit Logging** - Comprehensive activity tracking with partitioned storage
- **Forest Patrimony** - Multi-level hierarchical asset management
- **Biological Asset Management** - Forest asset tracking and measurements
- **Geospatial Processing** - Shapefile import with automated worker processing

### Advanced Features

  <Accordion title="Audit & Compliance">
    - Comprehensive audit logging for all actions
    - Partitioned audit tables for optimal performance
    - Detailed tracking of CREATE, UPDATE, DELETE, LOGIN, EXPORT, and VIEW actions
    - User activity monitoring and reporting
  </Accordion>

  <Accordion title="Background Processing">
    - Dedicated geospatial worker for shapefile processing
    - Asset measurement worker for biological asset calculations
    - Configurable batch sizes and polling intervals

  </Accordion>

## System Requirements

<Note>
  SMyEG requires Node.js 20 or higher and uses PostgreSQL 15+ for data storage. Docker is recommended for local development but optional for production deployments.
</Note>

| Component | Requirement |
|-----------|-------------|
| Node.js | 20.x or higher |
| PostgreSQL | 15.x or higher |
| Redis | 7.x or higher |
| Docker | Optional (recommended for dev) |
| PM2 | Optional (recommended for production) |
