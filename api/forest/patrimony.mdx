## Overview

The Forest Patrimony API manages the hierarchical structure of forest assets across four levels:

- **Nivel 2**: Properties (FINCA, PREDIO, HATO, FUNDO, HACIENDA, ABRAE)
- **Nivel 3**: Compartments (COMPARTIMIENTO, BLOCK, SECCION, LOTE, ZONA, BLOQUE, ZONIFICACION)
- **Nivel 4**: Management units (RODAL, PARCELA, ENUMERATION, UNIDAD_DE_MANEJO, CONUCO, OTRO_USO)
- **Nivel 5**: Plot references (REFERENCIA, SUBUNIDAD, SUBPARCELA, MUESTRA, SUBMUESTRA)

All endpoints require authentication and appropriate permissions (`forest-patrimony:READ`, `forest-patrimony:CREATE`, `forest-patrimony:UPDATE`, `forest-patrimony:DELETE`).

## Create Patrimony Record

<CodeGroup>
```bash POST /api/forest/patrimony
curl -X POST "https://api.smyeg.com/api/forest/patrimony" \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "level": "2",
    "data": {
      "code": "N2-001",
      "name": "Finca El Paraíso",
      "type": "FINCA",
      "totalAreaHa": 1500.5,
      "legalStatus": "ADQUISICION",
      "isActive": true
    }
  }'
```
</CodeGroup>

## Update Patrimony Record

<CodeGroup>
```bash PATCH /api/forest/patrimony
curl -X PATCH "https://api.smyeg.com/api/forest/patrimony" \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "level": "2",
    "id": "123e4567-e89b-12d3-a456-426614174000",
    "data": {
      "name": "Finca El Nuevo Paraíso",
      "totalAreaHa": 1550.0
    }
  }'
```
</CodeGroup>

### Request Body

<ParamField body="level" type="string" required>
  Hierarchy level: `"2"`, `"3"`, `"4"`, or `"5"`
</ParamField>

<ParamField body="id" type="string" required>
  UUID of the record to update
</ParamField>

<ParamField body="data" type="object" required>
  Partial update object (all fields optional, at least one required)
</ParamField>

### Special Behavior

- **Nivel 4**: Updating `isActive` triggers geometry updates in `forest_geometry_n4` table
- **Nivel 5**: Area is automatically recalculated when shape or dimensions change
- **Nivel 4**: Enqueues background recalculation job after update

### Response

Returns the updated patrimony record.

## Delete Patrimony Record

<CodeGroup>
```bash DELETE /api/forest/patrimony
curl -X DELETE "https://api.smyeg.com/api/forest/patrimony" \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "level": "2",
    "id": "123e4567-e89b-12d3-a456-426614174000"
  }'
```
</CodeGroup>

### Request Body

<ParamField body="level" type="string" required>
  Hierarchy level: `"2"`, `"3"`, `"4"`, or `"5"`
</ParamField>

<ParamField body="id" type="string" required>
  UUID of the record to delete
</ParamField>

### Cascade Protection

- **Nivel 2**: Cannot delete if Nivel 3 children exist
- **Nivel 3**: Cannot delete if Nivel 4 children exist
- **Nivel 4**: Cannot delete if Nivel 5 children exist; also deletes related geometry
- **Nivel 5**: Can delete freely

### Response

<ResponseExample>
```json
{
  "message": "Nivel 2 eliminado"
}
```
</ResponseExample>
