## Overview

The Forest Configuration API provides CRUD operations for various catalog entities used throughout the forest management system:

- **Species** (`/api/forest/config/species`): Plant taxonomic data
- **Land Use Types** (`/api/forest/config/land-use-types`): Land classification categories
- **Management Schemes** (`/api/forest/config/management-schemes`): Forest management approaches
- **IMA Classes** (`/api/forest/config/ima-classes`): Increment classification
- **Inventory Types** (`/api/forest/config/inventory-types`): Inventory methodologies
- **Product Types** (`/api/forest/config/product-types`): Harvested product specifications
- **Vegetal Materials** (`/api/forest/config/vegetal-materials`): Planting material catalog
- **Provenances** (`/api/forest/config/provenances`): Geographic origins
- **Spacings** (`/api/forest/config/spacings`): Planting density configurations
- **Level 4 Costs** (`/api/forest/config/level4-costs`): Administrative cost tracking

All endpoints require `forest-config:READ`, `forest-config:CREATE`, `forest-config:UPDATE`, or `forest-config:DELETE` permissions.

## Common Endpoint Pattern

All configuration endpoints follow a consistent REST pattern:

- `GET` - List records with pagination
- `POST` - Create new record
- `PATCH` - Update existing record
- `DELETE` - Delete record by ID

## Species

### Create Species

<CodeGroup>
```bash POST /api/forest/config/species
curl -X POST "https://api.smyeg.com/api/forest/config/species" \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "code": "PIN-CAR",
    "scientificName": "Pinus caribaea",
    "commonName": "Pino caribeño",
    "genus": "Pinus",
    "family": "Pinaceae",
    "taxonomicOrder": "Pinales",
    "isActive": true
  }'
```
</CodeGroup>

#### Request Body

<ParamField body="code" type="string" required>
  Unique code (1-80 characters)
</ParamField>

<ParamField body="scientificName" type="string" required>
  Scientific name (1-255 characters)
</ParamField>

<ParamField body="commonName" type="string">
  Common name (max 255 characters)
</ParamField>

<ParamField body="genus" type="string">
  Taxonomic genus
</ParamField>

<ParamField body="family" type="string">
  Taxonomic family
</ParamField>

<ParamField body="taxonomicOrder" type="string">
  Taxonomic order
</ParamField>

<ParamField body="isActive" type="boolean" default="true">
  Whether the species is active
</ParamField>

<CodeGroup>

## Update Pattern

All configuration endpoints support partial updates via `PATCH`:

<CodeGroup>
```bash PATCH /api/forest/config/{resource}
curl -X PATCH "https://api.smyeg.com/api/forest/config/species" \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "id": "species-uuid",
    "commonName": "Pino del Caribe",
    "isActive": false
  }'
```
</CodeGroup>

**Request Body**: Include `id` (required) plus any fields to update (at least one required).
