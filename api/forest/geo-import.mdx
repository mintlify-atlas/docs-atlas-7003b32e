## Overview

The Geo Import API handles shapefile uploads for forest geometry (Nivel 4 polygons). It supports:

- ZIP upload containing `.shp`, `.shx`, `.dbf`, and `.prj` files
- Asynchronous background processing via worker
- Job status tracking and validation
- Automatic land use variation creation

Requires `forest-patrimony:CREATE`, `forest-patrimony:UPDATE`, or `forest-patrimony:READ` permission.

## Upload Shapefile

<CodeGroup>
```bash POST /api/forest/geo/import
curl -X POST "https://api.smyeg.com/api/forest/geo/import" \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -F "file=@nivel4_polygons.zip" \
  -F "variationDate=2024-01-15" \
  -F "variationNotes=Actualización masiva GIS"
```

```javascript JavaScript
const formData = new FormData();
formData.append('file', shapefileZip);
formData.append('variationDate', '2024-01-15');
formData.append('variationNotes', 'Actualización masiva GIS');

const response = await fetch('/api/forest/geo/import', {
  method: 'POST',
  headers: {
    'Authorization': `Bearer ${token}`
  },
  body: formData
});

const { jobId, status } = await response.json();
```
</CodeGroup>

### Request Body (multipart/form-data)

<ParamField body="file" type="file" required>
  ZIP file containing shapefile components (`.shp`, `.shx`, `.dbf`, `.prj`)
</ParamField>

<ParamField body="variationDate" type="string">
  Date for land use variation records (ISO 8601 format: `YYYY-MM-DD`)
</ParamField>

<ParamField body="variationNotes" type="string">
  Optional notes to attach to variation records
</ParamField>

### Shapefile Requirements

The ZIP archive must contain all required components:

- **`.shp`**: Main geometry file (required)
- **`.shx`**: Shape index file (required)
- **`.dbf`**: Attribute database (required)
- **`.prj`**: Projection definition (required)

Optional files (`.cpg`, `.sbn`, `.sbx`, etc.) are allowed but not validated.

### Response

<ResponseField name="jobId" type="string">
  UUID of the created import job
</ResponseField>

<ResponseField name="status" type="string">
  Initial job status (always `PENDING`)
</ResponseField>

<ResponseField name="message" type="string">
  Confirmation message
</ResponseField>

<ResponseExample>
```json HTTP 202 Accepted
{
  "jobId": "job-123e4567-e89b-12d3-a456-426614174000",
  "status": "PENDING",
  "message": "Archivo recibido. El worker procesará la importación en segundo plano."
}
```
</ResponseExample>

### Processing Flow

1. ZIP file is validated for required components
2. Job record is created in database with status `PENDING`
3. File is stored at `public/uploads/geo-imports/{jobId}.zip`
4. Worker endpoint is triggered asynchronously
5. Client receives immediate response with `jobId`
6. Use `GET /api/forest/geo/import/{jobId}` to check status

## Check Job Status

<CodeGroup>
```bash GET /api/forest/geo/import/{jobId}
curl -X GET "https://api.smyeg.com/api/forest/geo/import/job-123e4567" \
  -H "Authorization: Bearer YOUR_TOKEN"
```

```javascript JavaScript
const response = await fetch(`/api/forest/geo/import/${jobId}`, {
  headers: {
    'Authorization': `Bearer ${token}`
  }
});

const job = await response.json();
console.log(`Status: ${job.status}`);
```
</CodeGroup>

## Download Import Template

<CodeGroup>
```bash GET /api/forest/geo/import?format=csv
curl -X GET "https://api.smyeg.com/api/forest/geo/import?format=xlsx" \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -o template.xlsx
```
</CodeGroup>

### Query Parameters

<ParamField query="format" type="string" default="csv">
  Template format: `csv` or `xlsx`
</ParamField>

### Response

Returns a downloadable template file with these columns:

- `nivel2`: Nivel 2 code (e.g., `N2-001`)
- `nivel3`: Nivel 3 code (e.g., `N3-001`)
- `nivel4`: Nivel 4 code (e.g., `N4-001`)
- `usoactual`: Current land use (e.g., `BOSQUE`)
- `usoanterior`: Previous land use (e.g., `NO BOSQUE`)
- `fechavariacion`: Variation date (YYYY-MM-DD)
- `observaciones`: Notes

## Worker Endpoint (Internal)

The worker endpoint (`POST /api/forest/geo/import/worker`) is called automatically after upload. It processes the shapefile and updates job status.

**Security**: Protected by `x-worker-secret` header (env: `GEO_WORKER_SECRET`).
